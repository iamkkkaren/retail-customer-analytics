# Retail Customer Analytics

I built this project to practise working with a real retail dataset from end to end — from raw messy data to a dashboard a business could actually use.

## What I did

Started with 1M+ rows of transaction data from the UCL Online Retail dataset and NZ Stats. Cleaned and restructured it in Python, then split it into normalised tables.

Designed the database schema in SQL, created the tables, and imported the cleaned data. Then ran regression analysis to look at churn risk and K-means clustering to group customers. Built 4 Power BI dashboards to show the results — sales overview, customer segments, churn risk, and a revenue forecast benchmarked against NZ retail data.

## Tools

Python (pandas, numpy) · SQL · Power BI

## What I found

High-value customers who hadn't bought recently were the biggest revenue risk. Flagged them as the top retention priority and made three recommendations for the business around retention, return rates, and inventory planning.
